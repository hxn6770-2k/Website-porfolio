MyPlanner App

Developed a task management app to enhance productivity and organization.
Implemented user authentication and cloud storage for secure task data management.
Utilized React Native for cross-platform mobile development, ensuring accessibility on both iOS and Android devices.
Enhanced user experience with UI/UX best practices, including intuitive navigation and responsive design.
Incorporated features like priority tagging, deadline reminders, and collaborative task sharing.

Enhancements Made

  • Improved data security through user authentication and encrypted data storage.
  • Enhanced functionality by implementing a real-time collaborative feature, allowing multiple users to manage tasks simultaneously.
  • Optimized app performance by employing algorithmic techniques for efficient data retrieval and processing, including sorting and priority tagging.
  • Refined the UI/UX design to follow accessibility standards, ensuring the app is user-friendly for diverse audiences.

Enhancement Narrative: My Planner Application

I. Brief Description of the Artifact

The artifact I selected is the To-Do List App, an Android application developed to help users manage tasks efficiently. Originally created as part of a Mobile Architecture and Programming course, the app offers task management features such as adding, viewing, and clearing tasks, with a user-friendly design that enhances the user experience. This artifact demonstrates my ability to combine backend functionality with frontend design, emphasizing modular code architecture.

II. Justification for Inclusion

I included this artifact in my ePortfolio because it reflects my skills in algorithm design, data structures, and user interface development. By enhancing the app to include task priority and sorting algorithms, it provides a solid example of my ability to apply algorithmic principles to solve real-world problems, such as organizing tasks efficiently. The addition of these functionalities showcases my ability to handle data structures, particularly the manipulation and sorting of task data based on various attributes (like priority and due date). This artifact satisfies Category Two and Three: Algorithms - Data Structures and Database.

  • Specific Components:
  • The task priority system demonstrates my skills in sorting algorithms.
  • The enhanced user interface reflects my ability to design interactive and intuitive user experiences.
  • The integration of Android SDK and Java shows proficiency in mobile app development.

III. How the Artifact Was Improved

Several key enhancements were made to the application, focusing on integrating a task priority system, implementing a custom sorting algorithm, and connecting the app to a database. These improvements provide a more robust and flexible task management system.

  • Key Enhancements:
  • 1. Database Integration for Task Storage: I connected the application to a database, updating the task model to include fields for priority and deadline. This allows for efficient CRUD operations (Create, Read, Update, Delete) and supports dynamic task management.
  • 2. Task Priority System: Added a priority field to each task, enabling users to assign High, Medium, or Low priority. This was implemented through modifications in the task data model.
  • 3. Custom Sorting Algorithm: Developed a sorting algorithm that organizes tasks based on their assigned priority and deadline. This showcases my ability to apply algorithmic principles and ensures tasks are presented in a meaningful way.
  • 4. UI Enhancements: Updated the user interface to allow users to input task priority and deadline when creating or editing tasks. The task list view was improved to reflect sorted tasks, enhancing the overall user experience.

IV. Meeting the Course Outcomes

With these enhancements, I successfully aligned the artifact with the following course outcomes:

  • Outcome 1: Collaboration and Improvement of Algorithms: The implementation of the task sorting algorithm based on priority demonstrates my ability to develop and improve algorithms, enhancing task management functionality.
  • Outcome 2: Clear Presentation of Technical Concepts: The updated codebase, which includes the priority field and sorting logic, reflects my ability to clearly present complex technical concepts in both code structure and user experience.
  • Outcome 3: Application of Algorithmic Principles: The new sorting algorithm based on both priority and deadlines showcases my application of algorithmic principles, improving the task management functionality.
  • Outcome 4: Use of Advanced Tools and Techniques: The integration of Android SDK, Java, and database systems demonstrates my proficiency in using advanced development tools.
  • Outcome 5: Mitigation of Software Design Risks: Enhancing the database schema and improving CRUD operations mitigated risks associated with poor data management, making the application more robust.

V. Reflection on the Process

This enhancement process was an invaluable learning experience. Integrating the database with the updated task model and sorting logic allowed me to explore new dimensions of backend development. One major insight was understanding the relationship between data structures and user experience, ensuring that database operations are both efficient and aligned with user needs.

  • Challenges Faced:
  • Database Integration: Updating the task model to support priority sorting required changes in the database structure, which presented challenges in ensuring smooth data storage and retrieval.
  • UI/UX Considerations: Modifying the user interface to support new functionality while keeping it intuitive required several iterations to balance functionality and simplicity.

This enhancement process deepened my understanding of how databases, algorithms, and UI design interact to create a well-rounded, functional application. The improvements made ensure that the app is more efficient in managing tasks and offers a smoother user experience. This process solidified my ability to design and implement scalable, robust solutions in mobile development, aligning with the course outcomes related to software design, data management, and algorithmic problem-solving.
