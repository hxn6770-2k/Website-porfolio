My Planner App

The artifact I selected is the To-Do List App, an Android application developed to help users manage tasks efficiently. Originally created as part of a Mobile Architecture and Programming course, the app offers task management features such as adding, viewing, and clearing tasks, with a user-friendly design that enhances the user experience. This artifact demonstrates my ability to combine backend functionality with frontend design, emphasizing modular code architecture.

Enhancement Plan

Justification for Inclusion

I included this artifact in my ePortfolio because it reflects my skills in algorithm design, data structures, and user interface development. By enhancing the app to include task priority and sorting algorithms, it provides a solid example of my ability to apply algorithmic principles to solve real-world problems, such as organizing tasks efficiently. The addition of these functionalities showcases my ability to handle data structures, particularly the manipulation and sorting of task data based on various attributes (like priority and due date). This artifact satisfies Category Two and Three: Algorithms - Data Structures and Database.

  • Specific Components:
  • The task priority system demonstrates my skills in sorting algorithms.
  • The enhanced user interface reflects my ability to design interactive and intuitive user experiences.
  • The integration of Android SDK and Java shows proficiency in mobile app development.

How the Artifact Was Improved

Several key enhancements were made to the application, focusing on integrating a task priority system, implementing a custom sorting algorithm, and connecting the app to a database. These improvements provide a more robust and flexible task management system.

  • Key Enhancements:
  • 1. Database Integration for Task Storage: I connected the application to a database, updating the task model to include fields for priority and deadline. This allows for efficient CRUD operations (Create, Read, Update, Delete) and supports dynamic task management.
  • 2. Task Priority System: Added a priority field to each task, enabling users to assign High, Medium, or Low priority. This was implemented through modifications in the task data model.
  • 3. Custom Sorting Algorithm: Developed a sorting algorithm to prioritize tasks based on user-defined criteria, allowing tasks to be sorted by deadline and priority level.

Each enhancement was tested thoroughly to ensure performance and reliability, leading to an application that not only meets user needs but also demonstrates my proficiency in software development practices.

Lessons Learned

Through the development of this application, I learned the importance of user feedback in guiding app design. Iterating based on user input has significantly improved the application's usability and functionality. I also enhanced my skills in database management and learned how to write efficient queries for optimal performance.
