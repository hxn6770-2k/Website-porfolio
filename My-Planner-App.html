To Do List App

The artifact I have selected is the To-Do List Application, originally developed for a Mobile Architecture and Programming course. It is an Android app designed to help users manage tasks efficiently, with features like adding, viewing, and clearing tasks. In this enhancement, I focused on integrating a task priority system and improving the task sorting functionality by connecting the app to a database. The database now supports sorting tasks by both priority and deadline, resulting in a more robust and flexible task management system. The enhancements prioritize algorithmic efficiency and database optimization, ensuring that the application can handle growing data while maintaining responsiveness.

How the Artifact Was Improved

Several key enhancements were made to the application, focusing on integrating a task priority system, implementing a custom sorting algorithm, and connecting the app to a database. These improvements provide a more robust and flexible task management system.

  • 1. Database Integration for Task Storage:
    • The application was connected to a SQLite database, updating the task model to include fields for priority and deadline. This enabled efficient CRUD operations (Create, Read, Update, Delete) for tasks, ensuring that the app can manage task data persistently across sessions.
    • By storing both priority and deadline in the database, I improved the app's ability to manage tasks dynamically, ensuring efficient data storage, retrieval, and manipulation. The database queries were optimized to reduce the time complexity of data operations (e.g., retrieval or updates), ensuring minimal latency during task management.
    • Optimization: Efficient use of indexed columns for priority and deadlines minimized retrieval time, ensuring that the sorting of tasks by these fields was not bottlenecked by slow database queries.
  • 2. Task Sorting by Deadline and Priority:
    • I developed a custom sorting algorithm that orders tasks by priority (High, Medium, Low) and deadline (earliest due first). The algorithm was designed with a time complexity of O(n log n) (using a hybrid of Merge Sort and database indexing), which ensures that sorting remains efficient even as the number of tasks increases.
    • The sorting logic was dynamically linked to the database, so tasks are automatically sorted based on the latest information stored in the database. This tight integration ensures that changes in task data—whether priority or deadline—are reflected immediately in the task display without any noticeable delay.
    • Algorithmic Complexity: Sorting tasks involves using a stable sort to maintain order, and its O(n log n) efficiency ensures scalability as the number of tasks grows.

    Here is the updated code for adding a priority field to each task, enabling users to assign High, Medium, Low priority. This was implemented through modifications in the task data model.

    Developed a sorting algorithm to prioritize tasks based on user-defined criteria, allowing tasks to be sorted by deadline and priority level. 

    
                                // Sorting tasks based on priority and deadline
                                public List sortTasks(List tasks) {
                                    Collections.sort(tasks, (task1, task2) -> {
                                        // First compare by priority
                                        int priorityComparison = task1.getPriority().compareTo(task2.getPriority());
                                        if (priorityComparison != 0) {
                                            return priorityComparison;
                                        }
                                        // If priority is the same, compare by deadline
                                        return task1.getDeadline().compareTo(task2.getDeadline());
                                    });
                                    return tasks;
                                }

The UI remains responsive due to the efficient algorithmic design of the backend, ensuring smooth sorting even when managing large task lists.
